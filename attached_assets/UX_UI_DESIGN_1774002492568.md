### Left Panel Components

**Template Selector (dropdown):**
- Full-width dropdown at top of panel
- Each option shows: template name + small aspect ratio indicators
- On selection: updates recommended asset types filter, loads Layout Spec preview
- "Custom (freeform)" option at bottom

**Asset Picker:**
- Search input (text, filters by name/tags)
- Filter pills row: game tags (Crown U, Rumble U, etc.), asset type (character, gameplay, logo, background)
- Compact grid: 3 columns of ~60x60px thumbnails
- Click = select as primary (blue border, "Primary" label)
- Shift+click = select as supporting (lighter border, "Supporting" label, max 2)
- Selected assets pinned at top of picker with role labels and × remove buttons
- Scroll for more assets, lazy load on scroll

**Reference URL Field:**
- Text input: placeholder "Paste URL for visual reference..."
- States:
  - Empty: just the input field
  - Pasting: input disabled, spinner, "Capturing page..."
  - Analyzing: thumbnail appears, spinner, "Analyzing reference..."
  - Ready: thumbnail + green "Analyzed ✓" badge + × remove button
  - Error: red text error message + "Try another URL" or "Upload screenshot instead" link
- "Or upload screenshot" link below — opens file picker

**Brief/Context Selector:**
- Dropdown: saved context cards (title + first-line preview)
- "Write custom brief" link: expands inline textarea (auto-grows to content)
- Multiple briefs can be selected (shown as chips/tags above textarea)

- 2-column responsive grid
- Gap: 16px between cards
- Cards scale to fill available width, maintaining aspect ratio proportions
- 4 cards for image-only campaigns (IG Feed, IG Story, Twitter, LinkedIn)

**Headline Overlay Editing:**
- The headline text composited onto the image is clickable
- On click: text becomes editable in a floating input overlaid on the image position
- On blur/Enter: sends `POST /api/campaigns/[id]/recomposite` with new text
- Compositing re-runs (near-instant, <500ms) — image updates in place
- NO Imagen API call — only re-composites the existing raw background

**Caption Inline Rewrite:**
- User highlights text in the caption area
- A floating toolbar appears with "✎ Rewrite" button
- Click Rewrite → small prompt input appears: "shorter", "more casual", "add trademark ™", etc.
- Submit → Claude rewrites just the highlighted portion → inserts replacement
- Cancel → dismisses without changes

**Refine All Input:**
- Positioned above the grid
- Text input: "Refine all variants: [____________________________]"
- Press Enter → re-runs Imagen for ALL variants with refinement appended to prompt
- Then re-composites each with existing headline text
- All cards show spinner simultaneously

**Duplicate Detection Banner:**
```

│ "Ohio State Character Reveal"                                    │

└─────────────────────────────────────────────────────────────────┘
```
- Appears at top of center panel, above the grid
- Yellow/amber left border (4px)
- Dismissible with × or "Dismiss and Proceed"
- "View Similar Campaign" opens that campaign in a new tab

### Right Panel Components

**Activity Feed:**
- Scrollable list of generation events
- Format: `[icon] [Operation] [time/duration]`
- Icons: ✓ green (complete), ⟳ blue animated (in progress), ✕ red (failed), ↻ amber (retrying)
- Failed items show [Retry] button inline
- Collapsible to just icons row when minimized

**Action Buttons (stacked vertically, full-width):**
- **Save Draft** — outlined button, always available
- **Submit for Review** — primary (accent blue), available when campaign has generated variants
- **Schedule** — primary, opens inline date/time picker
- **Download All** — outlined, generates structured ZIP
- **Remix** — outlined, only visible on saved/existing campaigns

**Cost Indicator:**
- Small muted text: "Est. cost: $0.42"
- Updates as each API call completes or is estimated
- Amber text if exceeds daily threshold

### Progressive Loading Sequence

1. **T+0ms:** Card outlines appear with platform labels, preview frame chrome, skeleton shimmer on image area. Caption area shows skeleton lines.
2. **T+1-2s:** Claude returns → captions populate with fade-in. Headline text appears in card header (not yet composited on image). Character counts shown.
3. **T+2-4s:** Imagen returns per variant → raw image appears with brief fade-in. Compositing runs immediately (<500ms) → composited image replaces raw with crossfade. Headline text now visible ON the image.
4. **T+4-6s:** All image variants complete. Activity feed shows all green checkmarks.
5. **T+15-60s (if video):** Video variant loads last. Progress bar shows estimated time. Play button activates when ready.

## 9. Loading, Empty, and Error States

### Loading States

**Campaign Studio — Generating:**
- Card outlines with skeleton shimmer (pulsing gradient from bg-surface to bg-surface-2)
- Caption area: 3 skeleton lines
- Activity feed: spinning indicators with operation names
- Generate button: disabled, shows spinner + "Generating..."

**Asset Library — Loading:**
- Grid of skeleton cards (6-12 depending on viewport)
- Search/filter bar visible and functional (shows "Loading assets..." text)

**Calendar — Loading:**
- Calendar grid visible with skeleton shimmer in cells
- Month navigation functional

### Empty States

**Campaign Studio — No campaigns:**
- Center panel shows: large icon (sparkle/wand), heading "Create your first campaign", subtext "Select a template and assets to get started", no CTA button (inputs are in the left panel)

**Asset Library — No assets:**
- Full-width: drag-and-drop upload zone, heading "No assets yet", subtext "Upload brand assets to get started. Drag files here or click Upload.", [+ Upload Assets] button centered

**Calendar — No scheduled posts:**
- Calendar grid visible, all cells empty
- Banner: "No posts scheduled. Create a campaign and hit Schedule to see posts here."

**Review Queue — No pending reviews:**
- Empty columns with faded text: "No pending reviews", "Nothing in review", etc.
- Subtle illustration or icon in each column

### Error States

**API Call Failed (per variant):**
- Red border on failed variant card
- Error icon (✕) + specific message: "Image generation failed: rate limit exceeded"
- [Retry] button on the card
- Other variants unaffected

**Network Error:**
- Toast notification (top right): "Connection lost. Changes may not be saved." with [Retry] action
- Auto-retry on reconnection

**Auth Error:**
- Redirect to login page with message: "Session expired. Please sign in again."

**Form Validation Error:**
- Inline red text below the field: "Brand name is required" / "Hex color must be 6 characters"
- Field border turns red
- Save button shows shake animation briefly

## 10. Component Inventory

### Buttons
- **Primary:** accent blue bg, white text, rounded-sm. Hover: darker blue.
- **Secondary/Outlined:** transparent bg, accent blue border + text. Hover: subtle blue bg.
- **Ghost:** transparent bg, secondary text. Hover: bg-hover.
- **Danger:** red bg, white text. Used only for delete confirmations.
- **Icon Button:** square, ghost style with icon. Hover: bg-hover.
- All buttons: height 36px (sm) / 40px (default) / 48px (lg). Disabled: opacity 0.5, cursor not-allowed.

### Inputs
- **Text Input:** bg-surface-2, border-default border, text-primary. Focus: border-focus (blue), subtle glow. Height 40px. Placeholder: text-muted.
- **Textarea:** Same styling, auto-grow or fixed height as specified.
- **Dropdown/Select:** Same styling as text input. Chevron icon right. Options in dark dropdown panel.
- **Search Input:** Same + magnifying glass icon left. Clear × button when has value.
- **Chip/Tag Input:** Text input that converts entries to removable chips. Used for: banned terms, tags, hashtags.
- **Color Picker:** Small square swatch + hex input. Click swatch opens color picker popover.

### Cards
- **Asset Card:** bg-surface, border-default, radius-md. Hover: shadow-md + border-hover.
- **Variant Preview Card:** bg-surface, radius-md. Contains platform preview frame.
- **Review Card:** bg-surface, radius-md. Draggable in Kanban.
- **Calendar Card:** compact, brand-colored left border (2px), radius-sm.

### Badges/Status
- **Status Badge:** small pill (text-xs, 4px 8px padding). Colors: success-green, warning-amber, error-red, info-blue.
- **Count Badge:** circular pill on nav items. Red if > 0.

### Modals
- **Confirmation Modal:** centered, bg-surface, shadow-lg, radius-lg. Title + message + Cancel/Confirm buttons.
- **Schedule Picker:** date picker + time picker inline. "Schedule" primary button.
- **Asset Detail:** slide-out panel from right (480px wide) or centered modal.

### Toast Notifications
- Bottom-right stack. Auto-dismiss after 5 seconds. Types: success (green icon), error (red icon), info (blue icon), warning (amber icon).
- [×] dismiss button. Max 3 visible at once.

### Skeleton Loading
- Pulsing gradient animation from bg-surface to bg-surface-2
- Applied to: image placeholders, text lines, cards
- Matches the shape of the content it replaces
