## 1. Executive Summary

SparqMake™ is a self-hosted, AI-powered social media content generation and management tool built specifically for Sparq Games' marketing team. It replaces the current manual workflow — where marketing assets are scattered across Figma, Google Drive, and Notion, posts are created and scheduled manually, and brand consistency depends on institutional knowledge in people's heads — with a single application that generates on-brand, multi-platform social media content from approved brand assets using AI.

The tool combines five AI model/service families: Anthropic Claude for text intelligence (captions, hashtags, copy), Google Imagen 4 Ultra for image generation, Google Veo 3.1 Generate for video generation, Google Gemini 3.1 Flash for visual analysis of reference URLs, and ElevenLabs for AI-generated music, sound effects, and voiceover on video content. A post-generation compositing layer overlays text, logos, and branded elements onto generated images — producing finished, professional social graphics rather than raw AI output with separate captions. All generation is guided by a persistent brand DNA configuration, reusable campaign templates with visual layout specifications, and an approved asset library — ensuring every output is on-brand without manual prompt engineering by the end user.

The application will be built by Tony Dye using Replit Agent 4 and hosted on Replit.

## 2. Problem Statement

### 2.1 Current Pain Points (sourced from Weekly PR Catch Up — 2026/03/19)

- **Asset governance failure:** Old/outdated art was posted to social media. No single source of truth for "approved for social" assets. Art lives in Figma sandbox, Google Drive folders, and individual machines with no clear status labeling.
- **Manual content creation:** Jenn Day is creating and posting all social content manually — selecting assets, writing captions, formatting for each platform, posting individually to Twitter/X, LinkedIn, and Instagram.
- **No scheduling infrastructure:** No social calendaring tool. Posts happen in real-time rather than being planned and queued. The team identified building a social calendaring tool as the #1 priority marketing tool to build.
- **Aspect ratio problem:** Assets need to be reformatted for each platform (Twitter 16:9, Instagram 1:1 and 9:16, LinkedIn 1.91:1). This is done manually or not at all.
- **Brand consistency depends on people:** What "on-brand" means for Crown U, Rumble U, and Mascot Mayhem lives in Tony's and Jeffrey's heads, not in a system. New team members have no way to produce consistent output without heavy oversight.
- **No feedback loop:** Lessons learned from what works (darker backgrounds, off-center compositions, certain caption styles) are not captured systematically. Each new content creation session starts from scratch.

### 2.2 What Exists vs. What's Needed

**Google Pomelli** was identified and demoed during the PR meeting. It impressed the team ("those are exactly the kind of things I need" — Jenn Day) but has critical limitations: no direct publishing, one campaign at a time, no multi-brand support, no asset library, no approval workflow, no scheduling, and cannot understand Sparq's multi-game brand landscape.

**DNA Studio (open source)** provides a useful architectural reference (Next.js, Prisma, BullMQ, model-agnostic LLM layer, social OAuth) but assumes single-brand URL crawling and lacks asset management, approval workflows, multi-aspect-ratio generation, and image/video generation capabilities.

**SparqMake™** combines the best ideas from both — Pomelli's "paste and generate" simplicity with DNA Studio's self-hosted, customizable architecture — purpose-built for Sparq's specific workflow.

| AI — Image | Google Imagen 4 Ultra (via Gemini API) | Static image generation at multiple aspect ratios |
| AI — Video | Google Veo 3.1 Generate (via Gemini API) | Short-form video clip generation |
| AI — Audio | ElevenLabs (via API) | Music generation, sound effects, voiceover for video content |
| AI — Visual Analysis | Google Gemini 3.1 Flash (via Gemini API) | Reference URL screenshot analysis |

### 4.2 Multi-Model Pipeline

```
User Input (template + asset + brief + optional URL)
        │
        ├──→ [If URL provided] ScreenshotOne → Gemini 3.1 Flash → Structured Analysis JSON
        │
        ▼
   Context Assembly Layer (application code)
        │
        │   Assembles: Brand DNA + Template Prompt + User Brief + 
        │   Reference Analysis + Selected Library Assets
        │
        ├──→ Claude API ──→ Platform-specific captions + headline text (returns ~1-2s)
        │
        ├──→ Imagen 4 Ultra API (parallel calls per aspect ratio) ──→ Raw background images (returns ~2.5s each)
        │
        ├──→ Veo 3.1 API (if video requested) ──→ Video clips with native audio (returns ~15-60s)
        │
        └──→ ElevenLabs API (if custom audio requested) ──→ Music/SFX track (returns ~5-10s)
        │
        ▼
   Compositing Layer (server-side, Sharp/Canvas + ffmpeg)
        │
        │   For images: Overlays headline text, logos, gradients onto raw Imagen output
        │   using template Layout Spec (font, size, position, color definitions)
        │
        │   For video: Merges ElevenLabs audio track with Veo video (ffmpeg)
        │
        ▼
   Campaign Studio (all variants displayed simultaneously)
        │
        ├──→ User reviews, refines inline, edits captions and overlay text
        │
        ├──→ Duplicate detection: flags if similar campaign exists within 30 days
        │
        ├──→ Submit for Review → Approval Queue
        │
        ├──→ Schedule → Content Calendar → BullMQ publishes at scheduled time
        │
        ├──→ Download All → Structured ZIP package (see Export Format Spec)
        │
        └──→ Remix → Pre-loads assets into new campaign with different template
```

### 4.3 Data Model (Core Entities)

**Brand** — Top-level brand configuration (Crown U, Rumble U, Mascot Mayhem, Sparq Games corporate)
- Brand DNA settings (colors, fonts, voice, trademark rules, hashtag strategy, negative prompts)
- Associated templates
- Associated asset library

**Asset** — An item in the brand asset library
- Type: `visual` (image) or `context` (text brief/instruction)
- Status: `uploaded` | `approved` | `archived`
- Metadata: uploader, upload date, tags, game association, usage count
- File reference (storage URL)

**Template** — Reusable campaign pattern
- Name (e.g., "Character Reveal," "Game Update," "Article Promotion")
- Imagen prompt prefix and negative prompt
- Claude caption instruction per platform
- **Layout Spec** (NEW): Visual overlay definitions for compositing — headline text zone (position, font, size, color, weight), logo placement (position, size, opacity), gradient overlay (type, colors, opacity), padding/margins. This is what makes generated images into finished social graphics.
- Recommended asset types
- Target aspect ratios
- Version history (for refinement tracking)
- Performance metrics (approval rates, refinement frequency)

**Campaign** — A single content creation session
- Associated brand and template
- Selected assets (visual + context)
- Reference URL (optional) + analysis JSON
- Generated variants (one per platform/aspect ratio), each with raw image + composited image
- Generated captions (one per platform)
- Headline/overlay text (the text composited onto images, separate from caption)
- Status: `draft` | `pending_review` | `approved` | `scheduled` | `published` | `rejected`
- Refinement log (every inline edit captured with timestamp)
- Reviewer comments
- Source campaign ID (populated if this campaign was created via Remix from another campaign)

**CampaignVariant** — Individual platform variant within a campaign
- Platform + aspect ratio
- Raw image URL (Imagen output, no overlays)
- Composited image URL (final image with text, logo, gradient overlays applied)
- Video URL (if applicable)
- Audio source: `veo_native` | `elevenlabs_music` | `elevenlabs_sfx` | `uploaded` | `muted`
- Audio file URL (if ElevenLabs generated or uploaded)
- Caption text (editable)
- Original AI-generated caption (for diff tracking)
- Headline overlay text (the text composited on the image)
- Schedule and publish status

**CalendarEntry** — A scheduled post
- Associated campaign variant
- Platform target (twitter, instagram_feed, instagram_story, linkedin)
- Scheduled datetime
- Published datetime (null until posted)
- OAuth account reference
- Publish status: `scheduled` | `publishing` | `published` | `failed`

**RefinementLog** — Captures every user edit for template improvement
- Campaign ID, template ID
- Edit type: `image_refinement` | `caption_edit` | `overlay_edit` | `approval` | `rejection`
- Original value, new value (for caption diffs)
- Refinement prompt text (for image refinements)
- Timestamp

**HashtagSet** — Reusable hashtag groupings
- Name (e.g., "Ohio State Buckeyes," "Crown U Launch," "Summer Campaign")
- Brand association
- Hashtags (array of strings)
- Category: `school_specific` | `campaign` | `seasonal` | `trending` | `evergreen`
- Usage count (how often this set has been used in campaigns)
- Claude uses these as a source library rather than inventing hashtags from scratch

## 6. Reference URL Analysis Pipeline

### 6.1 Purpose

When a user pastes a URL into the Campaign Studio, the system captures and analyzes the page to extract visual and tonal characteristics that should influence the generated assets. This is not brand extraction (Sparq knows its own brand) — it's creative reference gathering.

### 6.2 Pipeline Steps

1. **Capture:** ScreenshotOne API receives the URL, returns full-page screenshot(s) in viewport-sized segments (3-5 images covering the entire page). Full-page stitched capture ensures no content is missed, including lazy-loaded and below-fold content. ScreenshotOne's clean mode removes ads, cookie banners, and chat widgets.

2. **Analysis:** Gemini 3.1 Flash receives all screenshot segments in a single multimodal API call with a structured analysis prompt. It returns a JSON object containing: visual mood, color strategy, typography feel, layout patterns, composition notes, and relevance assessment for Sparq's use case.

3. **Context Injection:** The structured JSON analysis + the original screenshot images are injected into the context assembly for Claude (caption generation) and Imagen (image generation). Claude receives the textual analysis as additional brief context. Imagen receives both the analysis text in its prompt AND the screenshots as style reference images alongside the brand library assets.

### 6.3 Analysis Prompt Structure

The Gemini Flash prompt should produce a JSON response with these fields:
- `visual_mood`: Overall aesthetic description (e.g., "bold, high-energy, dark with neon accents")
- `color_strategy`: Dominant colors and how they're used
- `typography_feel`: Font style impressions and hierarchy
- `layout_patterns`: Compositional approach and structure
- `composition_notes`: Specific techniques worth emulating
- `relevance_to_sparq`: How elements could translate to Sparq's brand context

## 7. Template Refinement Loop (Evaluation + Recommendation System)

### 7.1 Purpose

Systematically capture feedback from generation usage and produce AI-recommended improvements to campaign templates, reviewed and approved by a human before application.

### 7.2 Data Collection (Built Into v1 Core)

Every user interaction that represents feedback is logged to the RefinementLog table:
- Image refinement prompts (what users type to modify generated images)
- Caption edits (diff between AI-generated and human-published text)
- Approval/rejection outcomes with comments
- Asset selection patterns (which assets are chosen per template)
- Generation-to-approval time

### 7.3 Analysis Engine

An invocable function (not a background agent) that can be triggered manually or on a schedule. Uses Claude to analyze accumulated refinement data for a specific template and produce structured recommendations.

**Input to Claude:** Structured analysis package including generation count, approval rates, most common refinement prompts with frequency, caption edit patterns, rejection reasons, current template prompt text, and current caption instructions.

**Output from Claude:** Structured recommendations with: specific prompt prefix changes (add, remove, modify), caption instruction changes, negative prompt additions, projected impact based on which common refinements each change would eliminate, and evidence citations linking each recommendation to the specific user patterns that drove it.

### 7.4 Review Interface

Template Refinement Review screen (within Brand Settings):
- Current template displayed side-by-side with recommended changes
- Each recommendation shows the evidence (refinement patterns and counts) that drove it
- Approve, reject, or modify each recommendation individually
- Rejected recommendations are noted to avoid re-suggestion
- Approved changes are versioned (rollback capability)

## 8. Additional Feature Specifications

### 8.1 Text-on-Image Compositing Layer

**Purpose:** Transform raw AI-generated images into finished, professional social graphics with text headlines, logos, gradients, and branded elements overlaid programmatically.

**Why this matters:** The most effective social graphics have text ON the image — a headline, character name, school name, logo watermark, CTA. Without compositing, SparqMake produces "nice AI images with separate captions." With compositing, it produces **finished social graphics that look like a designer made them.**

**Architecture:** Imagen generates a clean visual background/scene (the negative prompt explicitly excludes baked-in text). After Imagen returns, a server-side compositing step applies overlays defined by the template's Layout Spec. This separation means headline text can be edited without regenerating the entire image.

**Template Layout Spec (added to each template):**
```json
{
  "headline_zone": {
    "position": "lower_third",
    "alignment": "left",
    "max_width_percent": 80,
    "padding_px": 24,
    "font_family": "brand_condensed_sans",
    "font_weight": 800,
    "font_size_px": 48,
    "color": "#FFFFFF",
    "text_shadow": true,
    "max_lines": 2
  },
  "logo_placement": {
    "position": "bottom_right",
    "offset_px": 24,
    "max_height_px": 40,
    "opacity": 0.9
  },
  "gradient_overlay": {
    "type": "linear",
    "direction": "bottom_to_top",
    "color": "#000000",
    "start_opacity": 0.7,
    "end_opacity": 0.0,
    "height_percent": 40
  }
}
```

**Implementation:** Sharp (Node.js image processing) or HTML Canvas rendered server-side. The compositing function takes: raw Imagen output + Layout Spec + headline text + logo file → composited final image. Both the raw and composited versions are stored.

**Aspect ratio adaptation:** Layout Specs can define per-aspect-ratio overrides (e.g., headline position shifts for 9:16 Story format vs. 1:1 feed format). If no override exists, the system adapts the base layout proportionally.

### 8.2 Audio for Video Content

**Three-tier audio system for Veo-generated video clips:**

**Tier 1 — Veo Native Audio (default, no extra cost):** Veo 3.1 generates native audio with video — ambient sound, synchronized effects. This ships with every video automatically. Adequate for many social clips.

**Tier 2 — ElevenLabs Generation (when you want control):** Text-to-music for background tracks ("8-second high-energy esports intro, electronic, building tension, drop at 5 seconds"). Text-to-SFX for individual sound effects ("cinematic whoosh transition," "bass impact hit"). Generated audio is commercially licensed. Cost: ~$0.06-0.20 per generation depending on plan.

**Tier 3 — Upload Custom Audio (always available):** Accept MP3/WAV upload. User attaches their own licensed track, voiceover recording, or custom audio. File picker in the Campaign Studio's video variant card.

**Audio/video merging:** ffmpeg handles combining audio tracks with Veo video server-side. Single command, runs on Replit. Options: replace Veo native audio entirely, mix with Veo audio (adjustable volume levels), or mute all audio.

**Mute option:** Strip all audio for platforms where music might trigger content ID flags, or for content intended to be consumed silently (many social users scroll with sound off).

### 8.3 Content Repurposing (Remix)

**Purpose:** Allow reusing assets from an existing campaign with a different template/treatment without starting from scratch.

**Trigger:** "Remix" button available on any existing campaign — accessible from the Content Calendar, Review Queue, or when viewing a completed campaign.

**Flow:** Click Remix → system creates a new campaign with the same primary/supporting assets pre-loaded → Campaign Studio opens with template selector highlighted (current template shown but changeable) → user selects a different template → generates fresh variants with the same assets but different visual treatment and copy style.

**Data:** New campaign stores a `source_campaign_id` reference for tracking which campaigns were derived from others. Useful for understanding content reuse patterns.

**Use case:** Jenn posts a character reveal on Monday. On Thursday, she wants a "gameplay spotlight" post featuring the same character. Remix pre-loads the character asset; she selects the "Gameplay Spotlight" template; new campaign generates instantly.

### 8.5 Duplicate/Similarity Detection

**Purpose:** Prevent accidental content repetition as posting volume grows.

**Trigger:** Automatically runs when a campaign reaches generation stage.

**Logic:** Database query checking for existing campaigns with the same template ID AND the same primary asset ID created within the last 30 days. This is a simple SQL query, not an AI-powered semantic comparison.

**UI:** If a match is found, a dismissible warning banner appears at the top of the Campaign Studio: "This looks similar to a campaign from [date] — [View Similar Campaign] or [Dismiss and Proceed]." Does not block generation — just informs.

**Future enhancement (v2):** Could expand to Claude-powered semantic similarity checking of caption text, or perceptual hash comparison of generated images.

### 8.6 Hashtag Library

**Purpose:** Centralized, reusable hashtag sets that Claude draws from during caption generation, ensuring consistency and preventing use of banned/overused tags.

**Location:** Third tab in Asset Library (Visual Assets | Briefs & Context | Hashtag Library).

**How it feeds into generation:** When Claude generates captions, the brand DNA's hashtag strategy tells it how many hashtags to use per platform. The Hashtag Library provides the *specific hashtags* to choose from. The relevant hashtag sets (filtered by brand + category) are included in Claude's prompt as a reference list. Claude selects from these rather than inventing hashtags.

**Quick-save from Campaign Studio:** If a user manually types a new hashtag combination in a caption that works well, a "Save as Hashtag Set" action lets them add it to the library directly from the caption editor.

## 9. Social Publishing

### 9.1 Supported Platforms

| Platform | Post Types | Aspect Ratios | API |
|----------|-----------|---------------|-----|
| Twitter/X | Image + text, Video + text | 16:9, 1:1 | Twitter API v2 |
| Instagram | Feed (image + caption), Story (image/video) | 1:1, 4:5, 9:16 | Instagram Graph API (via Facebook) |
| LinkedIn | Image + text | 1.91:1, 1:1 | LinkedIn Marketing API |

### 9.2 Dual Publishing Mode

**OAuth Publishing (primary):** Connect social accounts via OAuth. Schedule posts from the Calendar. BullMQ processes the queue and publishes at scheduled times. Status tracking (scheduled → publishing → published → failed with retry).

**Manual Download (always available):** Every generated asset is downloadable at any time — individual variants, per-platform packages, or full campaign ZIP. Captions are included as a text file in the ZIP. This is the fallback if OAuth fails and the primary mode during initial setup before accounts are connected.

Build order: Manual download first (immediately useful), OAuth publishing second.

### 9.3 OAuth Account Management

Settings screen for connecting social accounts. Each connected account shows: platform, account name, connection status, token expiry. Reconnect flow for expired tokens. Multiple accounts per platform supported (e.g., @SparqGames and @CrownUGame on Twitter).

## 12. Non-Functional Requirements

- **Performance:** Campaign Studio should feel responsive. Captions appear in <2s, images in <5s per variant. No full-page loading states — progressive rendering as each API call returns.
- **Reliability:** If any single API call fails, it fails gracefully with a retry button on that specific variant. Other variants are unaffected.
- **Security:** API keys stored as Replit Secrets (environment variables), never in code. OAuth tokens encrypted in database. Role-based access (admin can configure brand settings; editor can create/schedule; viewer can review).
- **Cost Tracking:** Every API call logs its estimated cost. Dashboard shows daily/weekly/monthly spend by model. Alert if daily spend exceeds configurable threshold.
- **Mobile:** Mobile-responsive for Calendar and Review Queue (check schedule, approve posts from phone). Campaign Studio is desktop-optimized — not degraded for mobile.

## 13. Open Questions & Future Considerations

- **SparqMake integration:** SparqMake™ generates content for daily social operations. SparqMake generates content at scale for 700+ universities via event-driven orchestration. Future bridge may exist where SparqMake outputs flow into SparqMake™'s calendar, but this is out of scope for v1.

- **A/B testing:** Generate multiple caption or visual variants for the same post, publish both, compare performance. Requires analytics integration first.
- **Multi-user simultaneous editing:** v1 assumes sequential use (one person in Campaign Studio at a time). Collaborative real-time editing is a future consideration.
