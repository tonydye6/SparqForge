## Journey 1: Jenn Creates a Character Reveal Campaign

**User:** Jenn Day (Marketing / PR — Editor role)
**Goal:** Create a multi-platform social media campaign announcing a new Crown U character for Ohio State.
**Starting State:** Logged in, Campaign Studio is open (default screen).

### Steps

**1. Select brand and template**
- Jenn selects "Crown U" from the Brand dropdown (top bar).
- She selects "Character Reveal" from the Template dropdown.
- The asset picker filters automatically to show Crown U approved assets. The template's recommended asset types (character_art, branded_background) are highlighted.

**2. Select assets**
- Jenn searches "Ohio State" in the asset picker search bar.
- She clicks the Ohio State Buckeyes character art → it's selected as Primary (blue border, "Primary" label pinned to top of picker).
- She shift-clicks a Crown U branded dark background → it's selected as Supporting (lighter border, "Supporting" label).

**3. Add brief (optional)**
- She selects "CLC Partnership Language Requirements" from the Brief dropdown.
- She also clicks "Write custom brief" and types: "This is the first Ohio State reveal — make it feel like a big moment. Emphasize the Buckeyes rivalry energy."

**4. Hit Generate**
- Jenn clicks [Generate ▶] in the top bar.
- **System actions (visible in Activity Feed):**
  - Duplicate check runs → no match found, no banner.
  - Context assembly: loads Crown U brand DNA + Character Reveal template + selected assets + brief + CLC context card + relevant hashtag sets (Ohio State Buckeyes set, Crown U Launch set).
  - Claude API call fires → returns in ~1.5s with JSON: captions per platform + headline text per platform.
  - Imagen API calls fire in parallel (5 calls: 1:1, 4:5, 9:16, 16:9, 1.91:1) → each returns in ~2.5s.
  - Compositing runs after each Imagen return: applies headline text, Crown U logo, gradient overlay per the Character Reveal Layout Spec.

**5. Progressive results appear**
- **T+1.5s:** Caption text populates in all 4 variant cards. Headline text appears in the card headers. Jenn can start reading captions immediately while images generate.
- **T+3s:** Instagram Feed (1:1) composited image appears — she sees the Ohio State character on a dark, dramatic background with "OHIO STATE ENTERS THE GAME" headline overlaid in bold white condensed text, Crown U logo in the bottom right, dark gradient making text legible.
- **T+4s:** Twitter (16:9) and Instagram Story (9:16) composited images appear.
- **T+5s:** LinkedIn (1.91:1) composited image appears. All variants complete. Activity feed shows all green checkmarks.

**6. Review and refine**
- Jenn reads the Twitter caption. It's 267 characters — under the 280 limit (shown in green). She likes it but wants the opener changed.
- She highlights "Ohio State just entered the chat." and clicks [✎ Rewrite].
- A small prompt input appears. She types "more dramatic, declarative statement" and submits.
- Claude rewrites just that portion → "The Buckeyes have arrived." replaces the highlighted text.
- She notices the Instagram Story headline text says "OHIO STATE ENTERS THE GAME" but she wants it shorter for the vertical format.
- She clicks the headline text on the Story card → it becomes editable → she types "BUCKEYES ARRIVED" → presses Enter.
- The system calls `POST /api/campaigns/[id]/recomposite` → compositing re-runs with new text on the existing raw image → the Story card updates in <500ms. No Imagen call needed.

**7. Save hashtags**
- She notices she manually added "#Buckeyes" to the Instagram caption. She highlights the hashtag combination "#CrownU #GoBuckeyes #Buckeyes #OhioState #CollegeSports" and clicks [Save as Hashtag Set].
- A small modal appears: name = "Ohio State Game Day", category = "school_specific". She saves it.

**8. Download and submit**
- She clicks [Download All] → a structured ZIP downloads with folders for each platform, containing composited images, raw images, caption.txt files, and metadata.json files.
- She clicks [Submit for Review] → campaign status changes to "pending_review" → card appears in Chase's Review Queue.

**End State:** Campaign submitted for review. ZIP downloaded to Jenn's machine. Hashtag set saved to library. Entire flow took ~4 minutes.

## Journey 4: Jenn Uses a Reference URL

**User:** Jenn Day (Editor)
**Goal:** Create a game update post that matches the visual style of a competitor's campaign she admired.
**Starting State:** Campaign Studio, fresh campaign.

### Steps

**1. Set up campaign basics**
- Selects "Crown U" brand, "Game Update" template.
- Selects a gameplay screenshot as primary asset.

**2. Paste reference URL**
- In the Reference URL field, she pastes: `https://riotgames.com/valorant/episode-8`
- **System actions:**
  - UI shows spinner: "Capturing page..."
  - ScreenshotOne API captures 3 viewport-sized segments of the full page (~2s)
  - Thumbnail of first screenshot appears in the URL field
  - UI shows: "Analyzing reference..."
  - Screenshots sent to Gemini 3.1 Flash with structured analysis prompt (~1.5s)
  - Flash returns JSON analysis: dark backgrounds, neon accent colors, condensed typography, high-contrast hero imagery
  - UI shows: thumbnail + "Analyzed ✓" green badge

**3. Generate with reference context**
- Jenn hits [Generate ▶].
- Context assembly injects the reference analysis into both the Imagen prompt (visual mood, composition techniques, color approach) and the Claude prompt (content tone, Sparq application notes).
- The reference screenshots are also passed to Imagen as additional style reference images alongside the selected library assets.
- Results: generated images incorporate the dramatic, high-contrast, dark aesthetic from the Riot Games page — but filtered through Crown U's brand DNA (Crown U colors, not Valorant colors; Crown U voice, not Riot's).

**4. Satisfied with results**
- The reference influence is visible: more dramatic lighting, stronger color contrast, condensed headline typography.
- Jenn makes minor caption edits and submits for review.

**End State:** Campaign generated with reference-influenced visual style. Reference analysis stored in campaign record for future reference.

## Journey 5: Tony Runs Template Refinement Analysis

**User:** Tony Dye (Admin)
**Goal:** Optimize the "Character Reveal" template based on accumulated usage data.
**Starting State:** Settings → Crown U → Templates.

### Steps

**1. Access refinement review**
- Tony clicks the Character Reveal template row → [Review Recommendations].
- If no analysis exists yet, he clicks [Run Analysis] to trigger it.

**2. System runs analysis**
- **System actions:**
  - Queries RefinementLog table for Character Reveal template, last 30 days.
  - Aggregates: 47 generations, 66% first-pass approval, most common refinements ("darker background" 11x, "more space for text" 8x, "character larger" 4x), caption edit patterns (Twitter captions shortened by avg 34 chars, "Get ready for" removed 6x).
  - Sends structured analysis package to Claude (Sonnet, temperature 0.3).
  - Claude returns JSON recommendations.

**3. Review recommendations**
- The Refinement Review screen shows two columns: Current (left) vs. Recommended (right).
- **Recommendation 1 — Imagen prompt:**
  - Change: ADD "Extremely dark, low-key background. Deep shadows dominate." to prompt prefix
  - Evidence: "darker background" refinement appeared in 23% of generations
  - Projected impact: ~11 fewer manual refinements per 47 generations
  - Tony clicks [Apply] ✓

- **Recommendation 2 — Imagen prompt:**
  - Change: MODIFY "Character should dominate the composition" → "Character should fill at least 60% of frame height, positioned slightly off-center"
  - Evidence: "character larger" refinement appeared in 9% of generations
  - Tony clicks [Modify] → adjusts to "Character should fill at least 50% of frame height" → applies

- **Recommendation 3 — Claude caption instruction (Twitter):**
  - Change: ADD "Open with a declarative statement, not a question. No 'Get ready for' openers. Keep under 250 characters to leave room for hashtags."
  - Evidence: "Get ready for" removed 6x, avg caption shortened by 34 chars
  - Tony clicks [Apply] ✓

- **Recommendation 4 — Claude headline instruction:**
  - Change: MODIFY max words from 6 to 4 based on headline edit patterns showing consistent shortening
  - Tony clicks [Apply] ✓

**4. Save as new version**
- Tony clicks [Save as New Version]. Template version increments to v4.
- Version history shows: v3 → v4 diff with all applied changes.
- v3 is available for rollback if v4 performs worse.

**End State:** Character Reveal template updated to v4 with evidence-based improvements. All future generations use the improved prompts.

## Journey 6: Jenn Remixes an Existing Campaign

**User:** Jenn Day (Editor)
**Goal:** Reuse the Ohio State character asset from Monday's reveal for Thursday's "Gameplay Spotlight" post.
**Starting State:** Content Calendar, viewing the week.

**2. Remix creates new campaign**

  - Same brand (Crown U)
  - Same primary asset (Ohio State character art)
  - Same supporting assets (Crown U dark background)

- Hits [Generate ▶]. Generation runs with same assets but "Gameplay Spotlight" template treatment.

**5. Submit**
- Jenn reviews, makes minor edits, submits for review.

**End State:** New campaign created from same assets with different template treatment. Original campaign's ID linked via sourceCampaignId for content reuse tracking.

## Journey 7: Tony Generates a Video Campaign with Custom Audio

**User:** Tony Dye (Admin)
**Goal:** Create a video campaign for Crown U with custom ElevenLabs-generated music.
**Starting State:** Campaign Studio, fresh campaign.

### Steps

**1. Set up campaign**
- Selects "Crown U" brand, "Character Reveal" template.
- Selects Ohio State character art as primary asset.
- Checks the "Include Video" option (checkbox near the Generate button or in template settings).

**2. Generate**
- Hits [Generate ▶].
- **System actions — images and video:**
  - Claude generates captions + headlines (parallel, ~1.5s)
  - Imagen generates 5 static image variants (parallel, ~2.5s each) + compositing
  - Veo 3.1 generates 2 video variants: landscape 16:9 and portrait 9:16 (sequential, ~30-45s each)
  - Video variants show progress indicator: "Generating video... ~30s remaining"

**3. Static images arrive first**
- Image variant cards populate with composited results.
- Video variant cards show progress bars.

**4. Videos arrive**
- After ~30-45s, the landscape video card shows a playback preview.
- Tony clicks play — the 8-second video shows the Ohio State character in a dynamic reveal with Veo's native audio (ambient game sounds, cinematic effects).
- The portrait video loads shortly after.

**5. Custom audio for landscape video**
- Tony likes the video but wants custom music instead of Veo's native audio.
- He clicks the Audio dropdown on the landscape video card → selects "Generate Music".
- A prompt input appears. He types: "8-second high-energy esports intro, electronic, building tension, bass drop at 5 seconds, Crown U championship feel"
- He clicks [Generate Music].
- **System actions:**
  - ElevenLabs API call: `POST /v1/text-to-music` with prompt and 8-second duration (~5-8s)
  - Audio file returns, stored in campaign storage
  - ffmpeg merge: strips Veo native audio, applies ElevenLabs track → new merged video stored
  - Variant updated: audioSource = "elevenlabs_music", audioUrl set, mergedVideoUrl set

**6. Preview merged result**
- The video card refreshes with the merged audio. Tony plays it.
- The esports intro music builds with the character reveal. It sounds professional and on-brand.

**7. Upload audio for portrait version**
- For the portrait video, Tony has a custom 8-second audio clip (a licensed track the team uses).
- He selects "Upload Audio" from the portrait video's Audio dropdown.
- File picker opens → he selects the MP3.
- **System:** uploads audio, ffmpeg merges with portrait video (replace mode), variant updated.

**8. Download and review**
- Tony clicks [Download All] → ZIP includes:
  - All 5 static image platform folders (composited + raw + captions)
  - video/ folder with landscape_16x9.mp4 (with ElevenLabs music) and portrait_9x16.mp4 (with uploaded audio)
- Submits for review.

**End State:** Full campaign with 5 static image variants + 2 video variants with custom audio. All stored, all downloadable, submitted for review.
