# SparqForge 30-Post Replit Test Calendar

## Purpose

This 30-post calendar is designed specifically to test the upgraded SparqForge build shown in the latest screenshot. It assumes these improvements are now present:
- asset intelligence backend with packet assembly
- asset intelligence frontend with role-based selection and grouped brand assets
- content plan layer with CSV import and plan-to-campaign conversion

Because of that, this calendar is built less like a pure marketing schedule and more like a structured product test suite for Sparq’s parent-brand content engine.

## What this calendar is testing

Across the 30 posts, the goal is to exercise:
- subject reference selection
- style reference selection
- context-card use
- compositing-only logo behavior
- template fitness
- content plan import
- plan-to-campaign conversion
- cross-platform adaptation
- recurring series consistency
- proof-heavy versus world-building content

Primary platforms across this calendar:

## Why this version is better for Replit testing

This calendar deliberately repeats template families and packet types so you can judge whether the new asset-intelligence layer is behaving consistently instead of only producing one-off wins.

It also intentionally tests:
- repeated use of Packet A for style/compositing-heavy posts
- repeated use of Packet B for founder-led posts
- repeated use of Packet C for context/diagram-heavy explainers
- repeated use of Packet D for subject + style interactions
- one clear Packet E case for partnership/proof compositing fidelity

## Recommended test sequence

Test these in order:
- 1
- 6
- 8
- 3
- 7
- 20
- 15
- 17

Then move into the rest of the 30 once the system behavior feels stable.

## Evaluation rules

For each generated post, capture:
- was the packet choice appropriate
- were the correct asset roles used
- did the output need major editing
- did the visual look on-brand
- did the caption fit the platform
- was the plan-to-campaign flow smooth
- should this asset combination be recommended again

## Final recommendation

Use this calendar as a product validation sequence first and a publishing calendar second. The more consistent the outputs become across repeated packet types, the more confident you can be that SparqForge is ready for higher-volume production.
